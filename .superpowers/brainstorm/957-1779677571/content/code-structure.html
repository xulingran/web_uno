两种模式，两套 Store

🏠 单机 / 房主模式
使用现有模块，几乎不改：
✅ gameStore.ts（权威状态）
✅ useGameEngine.ts（AI 驱动）
✅ 所有 UI 组件不变
房主额外职责
状态变化时 → stateView 过滤 → 通过 peerHost 广播给客户端
🎮 客户端模式
使用新的只读 store：
🆕 remoteGameStore（只读状态）
🆕 peerClient（发送操作）
✅ UI 组件复用（读取 store 接口一致）
客户端行为
玩家操作 → peerClient 发消息 → 房主处理 → 收到新状态 → 更新 remoteGameStore

UI 组件适配

统一 Store 接口
// 组件不关心数据来源，统一接口
type GameStoreView = {
players: PlayerView[]
discardPile: Card[]
currentColor: CardColor
currentPlayerIndex: number
direction: Direction
phase: GamePhase
// ... 其他公共字段
}
// gameStore 和 remoteGameStore 都满足这个接口
// UI 组件只需传入正确的 store 即可
现有 UI 组件（GameBoard、PlayerHand 等）只需改为读取 GameStoreView 接口，无需知道是单机还是联机。
