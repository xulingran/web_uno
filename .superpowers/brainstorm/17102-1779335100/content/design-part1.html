设计方案 Part 1：基础架构

页面级滚动策略 + 卡牌尺寸自适应体系

1.1 页面级滚动策略分离

当前问题：全局 overflow:hidden 导致设置页无法滚动
当前
html, body, #root { overflow: hidden }
所有页面都被禁止滚动
→
改造后
#root { height: 100% }
.game-layout { overflow: hidden }
.settings-layout { overflow-y: auto }
各页面独立控制滚动行为

具体做法

  • 移除 index.css 中 html, body, #root 上的 overflow: hidden
  • 在 GameBoard 组件根元素上保留 overflow-hidden（游戏页确实不需要滚动）
  • SettingsPage 的根元素改为 h-screen overflow-y-auto，确保内容超出时可滚动

1.2 卡牌尺寸自适应体系

通过 CSS 变量 + clamp() 实现卡牌尺寸随视口缩放
CSS 变量定义 
:root {
  --card-w: clamp(60px, 8vw, 90px);
  --card-h: clamp(90px, 12vw, 135px);
  --card-radius: clamp(6px, 0.8vw, 10px);
  --card-font-sm: clamp(10px, 1.2vw, 14px);
  --card-font-lg: clamp(24px, 3vw, 36px);
}
缩放效果示意
7
小屏
~60×90
7
中屏
~75×112
7
大屏
~90×135

具体做法

  • 在 index.css 或 tailwind.config.js 中定义卡牌相关的 CSS 变量
  • Card 组件：移除硬编码 90×135，改用 CSS 变量
  • CardBack 组件：三种尺寸（normal/top/side）都改为基于主卡牌尺寸的缩放比例
  • AI 手牌的 FanCards 容器尺寸也跟随 CSS 变量动态计算
  • 抽牌堆/弃牌堆的间距和容器尺寸同步调整

涉及的文件

  • src/index.css — 添加 CSS 变量
  • src/components/Card.tsx — 使用变量替代硬编码
  • src/components/CardBack.tsx — 同上
  • src/components/AIHand.tsx — FanCards 容器跟随缩放
  • src/components/DrawPile.tsx — 容器尺寸跟随
  • src/components/DiscardPile.tsx — 容器尺寸跟随
