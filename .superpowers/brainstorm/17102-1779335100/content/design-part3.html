设计方案 Part 3：设置页面 + 弹窗响应式改造

设置页可滚动 + 面板宽度适配 + 弹窗尺寸自适应

3.1 设置页面改造

解决：内容溢出屏幕无法滚动 + 面板过窄
当前 
SettingsPage 根元素:
  min-h-screen flex flex-col

内容区:
  flex-1 overflow-y-auto px-4 py-6
  └─ SettingsPanel (max-w-lg = 512px)

问题:
1. 根 #root overflow:hidden 阻止滚动
2. max-w-lg 在宽屏上太窄
3. 分组折叠后内容间距紧凑
改造后 
SettingsPage 根元素:
  h-screen flex flex-col overflow-y-auto

内容区:
  flex-1 overflow-y-auto px-4 sm:px-6 py-6
  └─ SettingsPanel
       max-w-lg sm:max-w-2xl mx-auto
       分组间距 gap-5 → gap-6

改进:
1. 根元素 h-screen + overflow-y-auto 允许滚动
2. 面板宽度响应式：小屏 lg，中屏以上 2xl
3. 分组内项目间距从 py-2 → py-2.5，增加呼吸空间
4. 分组边框对比度提升：border-white/15

具体做法

  • SettingsPage 根元素：h-screen 替代 min-h-screen，确保容器有明确高度
  • 内容区保留 overflow-y-auto，配合 Part 1 的全局 overflow 修复，滚动生效
  • SettingsPanel 宽度：max-w-lg sm:max-w-xl lg:max-w-2xl，三档响应式
  • 分组间距：gap-4 → gap-5 sm:gap-6
  • 行内间距：py-2 → py-2.5
  • 分组边框：border-white/10 → border-white/15，稍微提升辨识度
  • 折叠按钮：py-3 → py-3.5，增加点击区域

3.2 弹窗/模态框统一响应式规范

四个弹窗统一适配策略
NewGameModal
  • 预设按钮 flex-wrap 允许换行
  • 难度按钮组间距 gap-1 → gap-2
  • 规则标签区加 py-1 增加间距
  • 整体加 max-h-[85vh] overflow-y-auto
ColorPicker
  • 按钮尺寸改为响应式：w-16 h-16 sm:w-[72px] sm:h-[72px]
  • 间距 gap-4 sm:gap-5
  • 容器加 p-6 sm:p-8 内边距适配
Scoreboard
  • min-w-[340px] → min-w-[280px] sm:min-w-[340px]
  • 玩家列表加 max-h-[40vh] overflow-y-auto 防溢出
  • 内边距 p-6 sm:p-8
ChallengeModal
  • 按钮间距 gap-3 → gap-4
  • 内边距跟随统一规范

统一弹窗规范

  • 所有弹窗遮罩：fixed inset-0 z-50，bg-black/50 sm:bg-black/60
  • 内容容器：max-w-[90vw] sm:max-w-none，防止小屏溢出
  • 内边距：p-5 sm:p-8，两档响应式
  • 圆角：rounded-xl sm:rounded-2xl
  • 有长列表的弹窗加 max-h-[85vh] 限制高度
