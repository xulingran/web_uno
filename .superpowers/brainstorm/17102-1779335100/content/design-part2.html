设计方案 Part 2：GameBoard 布局重构

CSS Grid 布局 + AI 手牌区域优化 + 玩家手牌叠放策略

2.1 GameBoard 改为 CSS Grid 布局

当前：嵌套 Flexbox → 改为：语义化 Grid
当前 Flex 布局 
div.flex-col (全页)
  div.flex (GameInfo 顶栏)
  div.flex.justify-center (顶部AI)
  div.flex-1.flex (中行)
    div.w-28 (左AI) — 固定宽度
    div.flex-1 (中央牌堆)
    div.w-28 (右AI) — 固定宽度
  div.flex.justify-center (玩家手牌)
问题：固定宽度、flex-1 空间浪费、嵌套层级深
改造后 Grid 布局 
div.grid (全页, 3列 × 4行)
  grid-template-columns:
    auto 1fr auto
  grid-template-rows:
    auto auto 1fr auto

区域分配:
  [info]   跨3列  → GameInfo
  [top-ai] 跨3列  → 顶部AI行
  [left] [center] [right]
          左AI   牌堆区  右AI
  [hand]  跨3列  → 玩家手牌
优势：比例自适应、语义清晰、AI区域 auto 按内容撑开

具体做法

  • GameBoard 根元素从 flex flex-col 改为 grid，定义 3 列 4 行
  • 左/右 AI 列用 auto 宽度（按内容撑开），不再硬编码 w-28
  • 中央列用 1fr，自动占据剩余空间
  • GameInfo 行和玩家手牌行用 grid-column: 1 / -1 跨满
  • 顶部 AI 行同样跨满，内部用 flex 居中排列

2.2 AI 手牌区域优化

AI 手牌：动态尺寸 + 合理间距
顶部 AI
  • 多个 AI 时 gap 从固定 gap-6 改为响应式 gap
  • AI 卡牌背面的 top 尺寸跟随 CSS 变量
  • 容器加 overflow-x-auto 防止溢出
左右 AI
  • 旋转后的内容通过 padding 确保不溢出 Grid cell
  • side 尺寸卡牌背面跟随 CSS 变量
  • 无左右 AI 时 Grid 列折叠为 0

2.3 玩家手牌叠放策略

多牌时自动叠放，减少水平空间占用
当前：gap-1 固定间距
牌多时需水平滚动
→
改造后：负 margin 叠放
总宽度受控，无需滚动

具体做法

  • PlayerHand 中的卡牌间距改为动态计算：margin-left: -${overlap}px
  • 叠放量 = max(0, (cardCount * cardWidth - containerWidth) / (cardCount - 1))
  • 保留 overflow-x-auto 作为极端情况的兜底
  • hover 时被叠放的卡牌通过 z-index 提升并稍微上移
  • 容器宽度用 max-w-[95vw] 限制，保证不出屏

2.4 GameInfo 信息栏适配

  • 改为 flex-wrap，允许信息项在窄屏时自动换行
  • 分隔符 <div class="w-px h-5"> 在换行时隐藏（通过 hidden sm:block）
  • 整体加 px-3 sm:px-6 响应式内边距
